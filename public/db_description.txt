HOMD Database Architecture


The HOMD frontend code is based on open-source Node.js (a server-side 
JavaScript runtime environment) 
and Express.js, a web application framework that runs on top of Node.js.

Node.js is a free, open-source, cross-platform JavaScript runtime environment.

The HOMD database backend is based on MySQL and uses InnoDB tables for its data.
Database rules:
-Focus on trying to minimize redundancy in the database
-Maximize speed of queries.
-Each table is linked to other tables by either the HMT-ID, or GENOME-ID
   and the tables are indexed on certain fields depending on which parameters are expected to be searched.
   The indexing allows for faster queries.

One important point related to the database is that certain information
is queried from the database and placed in JSON format flat files
for the Node.js server to read on startup.
This means that the data is available in memory and much faster to access
than sql queries called during normal page loading. Some data is too
large to store in memory or too infrequently used to warrant keeping
it in memory so that SQL queries are used instead. 


The database is divided into four general sections:
Taxonomy, Genomes, Phage and Annotation tables which are linked to each other
using HMT-ID or the GENOME-ID
