HOMD Database Architecture


The HOMD frontend code is based on open-source Node.js (a server-side 
JavaScript runtime environment) 
and Express.js, a web application framework that runs on top of Node.js.

Node.js is a free, open-source, cross-platform JavaScript runtime environment.

The HOMD database backend is based on MySQL and uses InnoDB tables for its data.
Database rules:
-Focus on trying to minimize redundancy in the database
-Maximize speed of queries.
-Each table is linked to other tables by either the HMT-ID, or GENOME-ID
   and the tables are indexed on certain fields depending on which parameters are expected to be searched.
   The indexing allows for faster queries.

One important point related to the application is that certain information
is queried from the database and placed in JSON format flat files
for the Node.js server to read on startup.
This means that the data is available in memory and much faster to access
than sql queries called during normal page loading. Of course, some data is too
large to store in memory or too infrequently used to warrant keeping
it in memory so that SQL queries are used instead. 


The database is divided into four general sections:
Taxonomy, Genomes, Phage and Annotation tables which are linked to each other
using HMT-ID or the GENOME-ID

Overview of Software:
	The HOMD web application code uses a Linux (Ubuntu version 20+) operating system and Node.js (a server-side 
JavaScript runtime environment) with Express.js (the framework built on top of Node.js).
Node.js is a fast, lightweight, modular, cross-platform and open-source development environment which allows JavaScript to be run outside of the
web browser enabling full-stack application development, server creation, and system automation using a single programming language.

Express.js provides a layer of fundamental web application features such as Routing (HTTP methods such as GET and POST), 
Middleware (creates request and response objects) and the Request/Response Cycle.
	
	To make the app more responsive much of the data from the backend database is pulled into
JavaScript Object Notation (JSON) structured flat files that are read into memory when the application is started.
These JSON files are created by python scripts and pre-placed in the directory structure where they are 
available when the Node.js server is started (or re-started). These JSON objects that are now in memory are much faster to access
than SQL queries to the database providing a more responsive user experience.  Of course, some data is too
large to store in memory or too infrequently used to warrant keeping
it in memory so that occasional SQL queries are used instead. 
 

Database Structure:
	The HOMD backend database storage uses a MySQL relational format with standard InnoDB structured tables. The data tables are
roughly divided into four areas: Taxa, Genomes, Annotations and Viruses (Phage) and are linked together using taxon IDs
and genome IDs helping to minimize data duplication. The database tables themselves are indexed to facilitate sql queries. 
The overall database schema (PNG Image) and table structures (MySQL dump file) can be viewed and downloaded from the HOMD web site.
